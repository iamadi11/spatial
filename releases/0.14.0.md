# Release v0.14.0 — 2026-04-04

## New Rule: `unvirtualized-list`

**Factory**: `createUnvirtualizedListRule(threshold = 50)`
**File**: `src/rules/unvirtualized-list.ts`
**SOT**: Section 4.2.2, 7, 12

### What it detects

A node whose immediate children contain more than `threshold` siblings of the same component type — the hallmark of an unvirtualized list (e.g. 100× `<ListItem>` without `react-window` or `react-virtual`).

### How it works

1. Counts occurrences of each child `type` using a `Map<string, number>` — O(n)
2. Finds the dominant type (max count) in a second O(n) pass over the map
3. If `maxCount > threshold` → returns a `warning` issue on the parent node

### Why this matters

Rendering hundreds of same-type siblings causes significant layout and paint cost. This rule surfaces the pattern before code ships.

### Properties

- Pure function — no side effects, no DOM, deterministic
- Metrics unused — structural check only
- Threshold configurable (default: 50)
- Issue message includes actual count, dominant type, and threshold

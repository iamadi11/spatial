# Release 0.6.0 — 2026-04-03

## What's New

### [016] Nesting depth detection rule

Added two exports in `src/rules/nesting-depth.ts`:

- **`computeMaxDepth(root: ComponentNode): number`** — pure O(n) DFS utility that computes the maximum depth of a component tree (root = depth 1)
- **`createNestingDepthRule(threshold?: number): NestingDepthRule`** — tree-level checker that returns a `PerformanceIssue` when `computeMaxDepth(root) > threshold`, or `null` otherwise

Addresses "Nested layouts" and "deeply nested components" edge cases in SOT Section 4.2.2.

**Architecture note**: Uses a tree-level `check(root)` interface rather than the per-node `Rule` interface to preserve O(n) complexity. The existing engine and type contracts are unchanged.

- Default threshold: 10
- Severity: `warning`
- Deterministic: same tree always produces the same result

# Release 0.27.0 — 2026-04-17

## New Rules

### `render-prop-overload`
Flags component nodes that accept more than 3 non-event-handler function props.

**Why it matters**: Components with many render props (`renderHeader`, `getLabel`, `formatRow`, `transformData`, etc.) suffer from responsibility creep. Each of these props creates a fresh function reference on every parent render, which defeats child memoization — even when `React.memo` or `useMemo` is applied to the child.

**What it catches**:
- Components that pass multiple render functions as props (render-props pattern overuse)
- Components acting as configurable "mega-components" that should be split

**What it ignores**:
- Event handler props (`onClick`, `onChange`, `onFocus`, `onBlur`, etc.) — these are handled by the `event-handler-count` rule
- Non-function props (strings, numbers, booleans, null)

**Threshold**: Default 3. Fires when the count of qualifying props **exceeds** 3 (i.e., 4+).

**Complexity**: O(1) per node — single pass over the props object.

# Release 0.8.1 — 2026-04-03

## What's Changed

### [019] Expanded style-complexity expensive property detection

The `EXPENSIVE_PROPERTIES` set in `src/rules/style-complexity.ts` now includes:

- `animation` — CSS animation triggers compositing layers on every frame
- `transition` — CSS transitions can force repaints during the transition duration
- `willChange` — hints the browser to create a compositing layer, carrying GPU memory overhead

**Previous set** (5 properties): `boxShadow`, `filter`, `backdropFilter`, `transform`, `clipPath`

**New set** (8 properties): above + `animation`, `transition`, `willChange`

No interface changes. The rule's name, output format, and behavior contract are identical.

This is a pure measurement accuracy improvement per SOT Section 12: "improved measurement accuracy of existing rules".

## Test Coverage

3 new tests (one per new property), 138 total passing.

# Release 0.17.0 — 2026-04-04

## New Rules

### `memo-candidate` (item 031)

Flags nodes that are strong `React.memo` candidates: they re-render frequently **and** have many direct children.

**Pattern detected**: A node that re-renders more than `renderCountThreshold` times while having more than `childrenThreshold` direct children is a compound performance signal — each re-render cascades to all children unnecessarily.

**Behaviour**:
- Both conditions must be true simultaneously (AND logic)
- Default: `renderCountThreshold = 3`, `childrenThreshold = 3`
- Fires `warning` severity with message naming both counts
- Pure, O(1) — no tree traversal needed

**Usage**:
```ts
import { createMemoCandidateRule } from '@spatial/rules/memo-candidate'
registry.register(createMemoCandidateRule(3, 3)) // defaults
```

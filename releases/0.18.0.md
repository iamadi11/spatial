# Release 0.18.0 — 2026-04-04

## New Rules

### `single-child-chain` (item 032)

Detects "wrapper hell" — long chains of components where each wraps exactly one child without branching or adding structure.

**Pattern detected**: Consecutive nodes where each has at most 1 child form a single-child chain. When this chain exceeds the threshold, it indicates over-abstraction or unnecessary wrapper components that add reconciliation overhead.

**How chain length is counted**:
- Contiguous nodes with ≤1 child (wrappers + terminal leaf)
- A node with 2+ children resets the chain (not counted itself)
- Chain length = number of wrapper nodes + 1 (the leaf)

**Behaviour**:
- Default threshold: 4 (chain of 5+ nodes triggers)
- Tree-level rule — checked once against the root (same as `nesting-depth`)
- `nodeId` in the issue points to the root of the longest chain
- Metrics unused — structural check only
- O(n) traversal

**Usage**:
```ts
import { createSingleChildChainRule } from '@spatial/rules/single-child-chain'

// Tree-level rule — call .check() directly on the root
const issue = createSingleChildChainRule(4).check(root)
```

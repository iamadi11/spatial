# Release 0.10.0 — 2026-04-03

## What's New

### [024] SpatialProvider

Added `createSpatialHandler()` and `useSpatial()` in `src/adapters/index.ts`.

This is the final integration layer — combining the React fiber adapter (022), the metrics adapter (023), and the core engine into a single ≤3-line consumer API.

**`createSpatialHandler()`**
- Returns a render callback closure
- Each call increments renderCount, extracts the fiber tree via `extractTree()`, collects metrics via `collectMetrics()`, runs `analyze()` with all 8 built-in rules, and posts the `PerformanceResult` to `globalThis.__SPATIAL__`
- Production guard: returns a no-op function, zero overhead

**`useSpatial(): PerformanceResult | null`**
- Reads `globalThis.__SPATIAL__?.result`
- Returns `null` before first render or in production

**Usage in a React app:**
```tsx
// App entry point (3 lines):
import { createSpatialHandler } from 'spatial/adapters'
const handler = createSpatialHandler()
<React.Profiler id="spatial" onRender={handler}><App /></React.Profiler>
```

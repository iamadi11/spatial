# Release v0.15.0 — 2026-04-04

## New Rule: `anonymous-component`

**Factory**: `createAnonymousComponentRule()`
**File**: `src/rules/anonymous-component.ts`
**SOT**: Section 4.2.2, 7, 12

### What it detects

Components whose `type` string is:
- Empty (`""`)
- A known anonymous placeholder: `"Anonymous"` or `"Component"`
- ≤ 2 characters long (minified names like `"A"`, `"_c"`, `"C0"`)

HTML elements (lowercase type like `"div"`, `"section"`) are never flagged.

### Why this matters

Unnamed or minified component names cannot be profiled or debugged.
React DevTools displays them as `<Anonymous>` and engine issue messages
cannot point engineers to the right component. Adding `displayName` before
shipping resolves this.

### Properties

- Pure function — no side effects, no DOM, deterministic
- Metrics unused — structural (type name) check only
- No configuration required — anonymous name patterns are well-defined
- Issue message prompts engineer to add `displayName`

13 test cases: 4 happy path, 5 edge cases, 1 failure case, 2 determinism/unknown cases.
All 236 engine tests pass.

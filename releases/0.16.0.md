# Release 0.16.0 — 2026-04-04

## New Rules

### `boolean-prop-overload` (item 030)

Flags components whose props contain more boolean values than a threshold (default: 5).

**Pattern detected**: "Do-it-all" components that accumulate toggle flags (`isActive`, `isDisabled`, `isLoading`, `isReadOnly`, etc.) over time. Each boolean prop is a distinct re-render trigger.

**Behaviour**:
- Counts `typeof val === 'boolean'` across top-level props only
- Triggers a `warning` when count exceeds the threshold
- Metrics not used — structural check only
- Pure function, no DOM, deterministic

**Usage**:
```ts
import { createBooleanPropOverloadRule } from '@spatial/rules/boolean-prop-overload'
registry.register(createBooleanPropOverloadRule(5)) // default threshold: 5
```
